# 模擬試題難度校準規範

依據對真實會考（2023-2024閱讀測驗）7篇文章的量化分析，加上對 reading_1 的實測修正結果，訂出以下可重複執行的標準流程。

## 結構規範（比照真實會考閱讀測驗）

- 全卷 43 題：
  - 第一部分單題（第1-19題）：詞彙/文法單題，1句題幹＋4選項
  - 第二部分題組（第20-43題）：6-8 組文章，每組2-5題
- 題組文章類型至少涵蓋：
  - 1 篇 practical（應用文：菜單/公告/廣告/時刻表），**必須包含至少一題需要「多步驟資訊整合」**（例如同時比對價格計算規則＋營業時間規則＋特殊公休規則，不能只是單一事實查找）
  - 2-3 篇 narrative（敘事故事）
  - 2-3 篇 informational（說明文，可為科普/社會議題/人物傳記）
  - 1 組 dual passage（兩篇對立觀點的論說文）
  - 1 組 cloze（故事式克漏字，4格文法/語意選擇）

## 量化難度目標（已用真題實測校準）

| 指標 | 目標值 | 依據 |
|---|---|---|
| 敘事/說明文類文章平均字數 | **約 230-245字** | 真題7篇實測平均 242字 |
| 平均句長 | **14-17字/句** | 真題實測平均 14.4字/句 |
| 每篇文章刻意保留的「hs7000」進階字 | **1-2個** | 真題每篇文章觀察到 1-2 個加註翻譯的進階字 |
| 全卷 hs7000 進階字佔比 | **約 3-5%** | reading_1 修正後實測 4.3% |
| 全卷「真正超綱字」（扣除功能詞/專有名詞/檢查腳本誤判後）| **趨近 0%** | reading_1 修正後實測 0% |

## 執行流程（每份模擬試題都要走完整流程，不能只憑感覺）

1. **草擬全卷**：43題，依上方結構規範分配文章類型，文章先寫到位（不要為了怕超綱而刻意簡化語言或縮短篇幅）
2. **跑字數/句長檢查**：
   ```
   node -e "... 計算 groupSec.passages 每篇 passage 的字數/句數 ..."
   ```
   若平均字數明顯低於230字，回頭延伸文章內容（加一段細節/例子/因果解釋），不要用湊字數的贅詞
3. **跑字彙安全檢查**：
   ```
   node server/scripts/check_exam_vocab.js server/data/<檔名>.json
   ```
   檢查「真正需要複核的超綱字清單」：
   - 已知的檢查腳本假警報（可忽略）：縮寫（don't）、不規則動詞（taken/became/meant）、雙子音比較級（bigger）、反身代名詞（himself等，若腳本未列入FUNCTION_WORDS）、-bly結尾的副詞（responsibly）
   - 真正的缺漏：直接用 Supabase 補進 `words` 表（tags: ['hs7000']），不要為了避開而改寫句子挖掉這個字——除非這個字本身就不適合（生僻/專有名詞）
4. **確認 JSON 合法**：`node -e "JSON.parse(...)"`
5. **人工核對**：題號連續不重複、answer index 對應正確選項、explanation 邏輯與最終文章內容一致

## 工具

- 字彙檢查腳本：`server/scripts/check_exam_vocab.js`（用法：`node server/scripts/check_exam_vocab.js <考卷json路徑>`）
- 已知的檢查腳本限制：字尾還原邏輯不處理不規則動詞（taken/became/meant等）、雙子音比較級（bigger→big）、-bly結尾副詞（responsibly→responsible）；這些出現在超綱清單裡不代表真的有問題，需人工複查是否為誤判

## 範例

`server/data/gsat_sim_2023_reading_1.json` 已依此規範完整校準過，字數從151字延伸到231.8字（真題基準242字），可作為後續9份的參考範本。
