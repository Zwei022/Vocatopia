# 角色收藏系統 — 候補角色名單

俄羅斯方塊出戰角色的候補設計庫。每隻角色都是「食物 + 英文單字」主題，
稀有度越高，食物越高級、技能越強（全程被動 > 一次性保險 > 連擊獎勵 > 被動小幅加成）。

實際資料寫在 `game/tetris/characters.js`（單一資料來源，供首頁角色欄、收藏頁、
遊戲技能三處共用）。本檔案只是設計草稿 / 候補清單，正式上線一隻才把它搬進
`TETRIS_CHARACTERS`。

## 商店 / 常駐卡池（`game/tetris/gacha.js`）

商店畫面（`#shop`）已上線常駐卡池，資料定義在 `GACHA_POOL`：

| 等級 | 角色 | 機率 | 重複轉換金幣 |
|---|---|---|---|
| 特獎 | 龍蝦 legendary | 1% | 800 |
| 二獎 | 壽司 mythic | 4% | 400 |
| 三獎 | 可麗露 epic | 15% | 150 |
| 銘謝惠顧（無角色） | — | 80% | 直接獲得 30 金幣 |

飯糰是帳號預設就擁有，鬆餅改走其他獎勵管道發放，兩者都**不**放進卡池抽獎名單。
未中獎（銘謝惠顧）不會產生角色，直接發 30 金幣安慰獎，機率總和維持 100%。

單抽 120 金幣／十抽 1000 金幣。抽到已擁有的角色會自動折算成金幣退還（不會產生無用的重複卡）。

**已知限制**：角色持有清單（`voca_owned_chars`）目前只存在本機 localStorage，
沒有跟 Supabase 同步（金幣有同步）。玩家換裝置登入時，金幣會保留但抽到的角色不會，
之後如果要跨裝置保留角色需要另外加 Supabase 欄位同步。

之後要擴充卡池（例如把候補清單裡的餃子/馬卡龍/麻糬等角色也放進去，或開新卡池），
只要在 `GACHA_POOL.entries` 加項目、機率總和維持 100% 即可，不用動抽卡邏輯本身。

**抽卡結果揭曉（2026-07-11 更新）**：卡片改成要點兩次才揭曉——第一次點擊依稀有度亮出
光環（銘謝惠顧黯淡無光／史詩可麗露大量紫光／神話壽司大量紅光／傳奇龍蝦超大量金光），
第二次點擊才翻牌顯示結果。卡片尺寸也放大一倍（88px → 176px）。卡背圖案改成 App 圖示
（`public/images/app_icon.webp`），不再是 🪄 emoji。抽卡跟開獎都配有音效（見下方音效章節）。

**保底機制（2026-07-11 新增）**：50 抽內必中神話以上（壽司/龍蝦其中之一），100 抽內
必中傳奇（龍蝦）。計數器存在 localStorage `voca_gacha_pity`（`sinceLegendary`/
`sinceMythicPlus`），中獎後對應計數器歸零；傳奇保底優先於神話保底（抽到傳奇時兩個
計數器一起歸零，因為傳奇本身也滿足「神話以上」）。機率跟保底進度都會顯示在商店的
「查看機率」彈窗跟卡池預覽區。

## 角色取得方式（`characters.js` 的 `acquireHint` 欄位）

收藏頁現在點擊任何角色（含未擁有/鎖定的）都能開啟詳細資料彈窗，只是未擁有時「出戰」
按鈕會換成「🔒 如何獲得」說明區塊：

| 角色 | 取得方式 |
|---|---|
| 飯糰 | 帳號預設擁有 |
| 鬆餅 | 完成一篇歷屆會考試題，正確率達 70% 以上即可解鎖（2026-07-11 從「寫完全部8份」簡化） |
| 可麗露 | 商店常駐卡池抽卡（三獎，15%） |
| 壽司 | 商店常駐卡池抽卡（二獎，4%，50抽保底神話以上） |
| 龍蝦 | 商店常駐卡池抽卡（特獎，1%，100抽保底傳奇） |

## 音效（`game/tetris/sfx.js`，2026-07-11 新增）

環境沒有 AI 音效/音樂生成工具可用，改用 Web Audio API 即時合成振盪器音效，
不需要外部音檔素材。涵蓋：移動/旋轉/落地/消行/技能施放/爆炸/清盤/答對/答錯/
遊戲結束/新紀錄/抽卡開獎（依稀有度不同音色）。開關讀取 localStorage
`voca_sfx_on`（目前沒有 UI 開關，預設開啟，之後可接到設定頁的音效開關）。

## 角色技能現況（2026-07-11 更新）

技能機制已從單一的「+N秒」擴充成五種類型，全部都在 `game/tetris/quiz.js`：

| 類型 | 角色 | 施放時機 | 效果 | 用後狀態 |
|---|---|---|---|---|
| `bonusSeconds` | 飯糰 | 只能在英文選擇題（計時題）進行中 | 當前題目 +N 秒 | 冷卻至下一輪計時題結束自動恢復 |
| `comboShield` | 鬆餅 | 被動，消行單字題答錯時自動觸發 | 保住連勝不被歸零（-30分仍照扣） | 每局限用 1 次，無法手動施放 |
| `choosePiece` | 可麗露 | 隨時可手動施放 | 跳出方塊表格，指定下一個方塊 | 封印，需連續答對 2 題計時題解封 |
| `bombPiece` | 壽司 | 隨時可手動施放 | 下一個方塊變成壽司炸彈（強制單格），落地炸開 3×3 範圍全部清除，+400分 | 封印，需連續答對 2 題計時題解封 |
| `clearBottom` | 龍蝦 | 隨時可手動施放 | 直接清空棋盤最底 2 行，不論是否被鎖住（含懲罰灰列） | 封印，需連續答對 3 題計時題解封 |

**消行單字題連勝加乘倍率**：連續答對消行快問（word quiz），第一次答對是 ×1（無加成），
連續第二次起才開始加乘 ×1.1、第三次 ×1.2，每多連勝一題 +0.1x，封頂 ×2.0。只套用在
答對的 +50 分，答錯的 -30 懲罰不受影響且會把連勝歸零（除非鬆餅護盾擋下）。

**鬆餅取得方式已變更**：不再透過卡池抽獎，改成寫完全部 8 份歷屆會考試卷
（2023~2026 閱讀+聽力，不含模擬試題）自動解鎖，追蹤資料存在 localStorage
`voca_gsat_done`。

**引擎擴充**：`game/tetris/engine.js` 新增 `setNextType()`（覆寫下一個方塊）、
`markNextAsBomb()` + 爆炸邏輯、`clearBottomRows()`，都有對應單元測試在
`engine.test.js`（跑 `node game/tetris/engine.test.js`）。

## 技能機制現況（重要）

目前 `game/tetris/quiz.js` 只實作了「+N 秒作答時間」這一種技能機制（`skill.bonusSeconds`），
是飯糰「從容一刻」的技能邏輯。**其餘設計草案中的技能類型（連擊獎勵、金幣加成、免死、
移除錯誤選項等）目前都還沒有對應的遊戲引擎邏輯**，只是文案構想。

5 隻已上線角色（飯糰/鬆餅/可麗露/壽司/龍蝦）全部先套用同一套「+N秒」機制，
依稀有度給不同秒數（10/12/15/18/20秒），確保卡片上寫的技能都是真的能觸發的效果，
不會出現「看起來有技能但實際上沒作用」的落差。之後要做連擊/金幣/免死等新機制，
需要先在 `quiz.js` 補上對應的遊戲邏輯。

## 擁有狀態

5 隻已上線角色目前只有飯糰在 `DEFAULT_OWNED_CHARS` 預設擁有清單中，其餘 4 隻
（鬆餅/可麗露/壽司/龍蝦）會出現在收藏頁但顯示鎖定狀態（🔒），因為目前還沒有
商店/抽卡等解鎖機制。如果要讓玩家能實際拿到這些角色，需要先決定解鎖方式
（例如：金幣商店購買、抽卡池、或任務獎勵）。

> 「待上線」= 使用者已核准設計方向、尚未生圖 / 寫進 characters.js。
> 「候補」= 純候補名單，尚未核准。

## 稀有度技能強度規則（設計基準，之後新增角色要參考）

- 🟤 **普通 common** — 被動小幅加成型（微幅減免懲罰 / 小額獎勵，無次數限制但效果很輕）
- 🔵 **稀有 rare** — 連擊／區間獎勵型（達成條件才觸發，或整局限一次的中等效果）
- 🟣 **史詩 epic** — 一次性強力保險型（整局限一次的高強度效果）
- 🌈 **神話 mythic** — 即時保命／高頻觸發型（比史詩更即時，或限兩次以上但效果稍弱）
- 👑 **傳奇 legendary** — 全程被動壓倒型（無次數限制、整局持續生效，數值明顯高於以下所有階級）
